第 1 章

初识 Claude Code

了解 Claude Code 的定位、核心能力和适用场景

🤔 Claude Code 是什么？

Claude Code 是 Anthropic 推出的 AI 编程助手 CLI 工具。它不是一个简单的代码补全工具，而是一个能理解项目上下文、执行复杂任务的智能开发伙伴。

核心能力对比

能力 传统 IDE Copilot Claude Code
代码补全 ✅ ✅ ✅
多文件理解 ❌ ⚠️ 有限 ✅ 完整项目
执行命令 ❌ ❌ ✅ Bash/Git
自动规划任务 ❌ ❌ ✅ Agent 模式

🎯 适用场景

📁 代码库探索

接手新项目时，快速理解代码结构、找到关键文件

🔍 Bug 修复

根据错误信息，自动定位问题并修复

✨ 功能开发

从需求描述到完整实现，包括测试和文档

🔄 批量重构

跨越多文件的代码重构，保持一致性

💡 工作方式

1

你描述需求

用自然语言告诉 Claude 你想做什么

→
2

Claude 规划任务

分析需求，拆解成可执行的步骤

→
3

执行与反馈

使用工具操作文件、运行命令，展示结果

💡 关键理解

Claude Code 的核心是工具调用（Tool Use）。它通过调用一系列工具（读文件、写文件、运行命令等）来完成任务，而不是仅仅生成代码建议。

第 2 章

核心工具系统

理解 Claude Code 的 30+ 工具及其协同工作原理

🔧 什么是工具（Tool）？

工具是 Claude Code 与外部世界交互的能力单元。每个工具都有特定功能，比如读取文件、执行命令、搜索代码等。Claude 根据任务需求，智能选择和组合工具。

📂 文件操作工具

FileRead 读取

读取文件内容，支持指定行范围

Read:0→{"file_path": "/path/to/file.ts"}
FileWrite 写入

创建新文件或覆盖已有文件

Write:1→{"file_path": "test.txt", "content": "..."}
FileEdit 编辑

精确修改文件中的特定内容

Edit:2→{"file_path": "...", "old_string": "...", "new_string": "..."}

🔍 代码搜索工具

Grep 搜索

在代码中搜索文本模式，支持正则

Grep:3→{"pattern": "function.*main", "path": "./src"}
Glob 文件匹配

按模式查找文件，如 **/*.tsx

Glob:4→{"pattern": "**/*.test.ts"}

⚡ 命令执行工具

Bash 需谨慎

执行 shell 命令，如 git、npm、pytest 等

Bash:5→{"command": "git status", "description": "Check git status"}

🤖 智能工具

Agent 高级

委派子任务给专门的 Agent 处理

WebSearch 网络

搜索网络信息，获取最新资料

WebFetch 网络

获取并解析网页内容

🔍 从源码看工具系统

根据 restored-src/src/tools/Tool.ts，每个工具都有统一的接口：

  • name：工具名称
  • description：功能描述（Claude 用它来决定是否调用）
  • parameters：参数 Schema（定义需要什么输入）
  • execute：执行逻辑

🎨 工具调用流程

你提问 → Claude 分析 → 选择工具 → 执行工具 → 获取结果 → 生成回复
                ↑___________________________________________↓
                                    (可能多次循环)
💡 学习要点

理解每个工具的适用场景和限制比记住语法更重要。例如：Grep 适合找代码，Glob 适合找文件，Bash 适合运行命令。

第 3 章

命令与技能

掌握 40+ 内置命令和自定义技能系统

⌨️ 什么是命令（Command）？

命令是 Claude Code 提供的快捷操作，以 / 开头。它们封装了常用工作流，让你不必每次都输入完整的自然语言描述。

常用命令分类

📝 代码操作

/commit 自动生成提交信息并提交代码
/review 审查当前代码变更（类似 PR Review）
/clear 清空对话历史
/compact 压缩对话历史，释放上下文空间

⚙️ 配置管理

/config 查看和修改配置
/init 创建 CLAUDE.md 项目文档
/memory 管理持久化记忆

🎯 工作模式

/plan 进入计划模式，先设计后执行
/fast 切换到快速模式（响应更快）
/help 查看所有可用命令

🎓 技能系统（Skills）

技能是可复用的能力模块。内置技能如 commit、review-pr 封装了完整的工作流。你也可以创建自定义技能。

内置技能示例

/commit

自动分析 git diff，生成符合规范的提交信息，并执行提交

分析变更 → 生成信息 → 执行提交

/review-pr

审查 GitHub Pull Request，提供代码建议

获取 PR → 分析代码 → 输出 Review

🔧 自定义技能（进阶）

创建自定义技能 
# 在 ~/.claude/skills/ 目录创建 custom-skill.md
---
name: deploy-vercel
description: Deploy current project to Vercel
---

# Deploy Workflow
1. Check if vercel CLI is installed
2. Run build to ensure no errors
3. Execute `vercel --prod` for production deployment
4. Return the deployment URL
💡 命令 vs 工具的区别
命令 用户主动触发（/开头） 封装完整工作流
工具 Claude 自动调用 基础能力单元
第 4 章

权限与安全

掌握权限模式、Approval 机制和安全配置

🛡️ 为什么需要权限控制？

Claude Code 可以执行 Bash 命令、修改文件、推送代码。如果没有权限控制，可能意外删除重要文件或执行危险操作。

三种权限模式对比

手动模式

默认
  • ✅ 每次工具调用都询问
  • ✅ 最安全，适合新手
  • ❌ 效率较低
claude

Auto Mode

推荐
  • ✅ Claude 自动判断安全性
  • ✅ 安全操作自动执行
  • ⚠️ 危险操作会阻断
claude --permission-mode auto

Full Auto

谨慎
  • ✅ 极少询问
  • ⚠️ 危险操作也会执行
  • ❌ 不适合生产环境
claude --permission-mode full-auto

⚙️ 配置文件详解

~/.claude/settings.json 
{
  "permissions": {
    "defaultMode": "auto",
    "allow": [
      "Bash:git status",
      "Bash:git diff",
      "Bash:pytest*",
      "Bash:npm install*",
      "WebSearch:*",
      "Read:*",
      "Write:*"
    ],
    "deny": [
      "Bash:rm*",
      "Bash:git push*",
      "Bash:sudo*",
      "Write:.env*"
    ]
  }
}

配置项说明

配置项 说明 示例
defaultMode 默认权限模式 "manual", "auto", "full-auto"
allow 自动允许的工具/命令 ["Bash:*", "Read:*"]
deny 始终拒绝的操作 ["Bash:rm*"]

🔐 危险操作识别

🗑️
删除操作 rm -rf 会永久删除文件
📤
强制推送 git push --force 会覆盖远程历史
🔑
敏感文件 .env、config.json 包含密钥
💡 最佳实践
  1. 新项目先用 手动模式，了解 Claude 的行为
  2. 熟悉后切换到 Auto Mode，平衡效率和安全
  3. 生产环境慎用 Full Auto
  4. 用 deny 配置保护敏感操作

🤖 Agent 模式详解

Agent 模式让 Claude 能够自主规划并执行多步骤任务，而不是每步都问你。它会拆解任务、并行执行、自动纠错。

Agent vs 普通模式对比

普通模式

你：改这个文件
↓
Claude：改好了，下一步？
↓
你：改那个文件
↓
Claude：改好了...

❌ 10 个文件 = 10 轮对话

VS

Agent 模式

你：重构这10个文件
↓
Claude Agent
分析依赖 并行修改 运行测试
↓
✅ 全部完成！

✅ 10 个文件 = 1 轮对话

🪝 Hooks 系统

Hooks 让你在特定事件发生时自动执行操作，实现真正的自动化工作流。

可用 Hooks

SessionStart

每次启动 Claude Code 时触发

FileWrite

写入文件后触发

FileEdit

编辑文件后触发

UserPromptSubmit

用户发送消息前触发

Hook 配置示例

自动格式化 + 测试 
{
  "hooks": {
    "FileWrite": [
      {
        "matcher": "*.py",
        "hooks": [
          {
            "type": "command",
            "command": "black $CLAUDE_FILE && pytest $CLAUDE_FILE -x"
          }
        ]
      }
    ]
  }
}

🎯 Coordinator 多 Agent 协调

复杂任务可以委派给多个专门的 Agent，它们并行工作，Coordinator 负责协调结果。

Coordinator

搜索代码
Edit Agent
修改文件
Test Agent
运行测试
Review Agent
审查结果

📋 案例 1：新项目接入配置

配置类

为团队创建标准化的 Claude Code 配置

1

创建项目级配置

在项目根目录创建 .claude/CLAUDE.md：

# 项目规范

## 技术栈
- React + TypeScript
- Tailwind CSS
- Jest 测试

## 代码规范
- 使用函数组件
- Props 必须定义类型
- 组件必须写测试

## 提交规范
- feat: 新功能
- fix: Bug 修复
- refactor: 重构
2

配置权限

编辑 ~/.claude/settings.json，允许该项目的测试命令：

{
  "permissions": {
    "allow": [
      "Bash:npm test*",
      "Bash:npm run build"
    ]
  }
}

🔄 案例 2：自动化开发工作流

自动化

保存文件时自动格式化 + 测试

~/.claude/settings.json 
{
  "permissions": {
    "allow": [
      "Bash:prettier*",
      "Bash:eslint*",
      "Bash:jest*"
    ]
  },
  "hooks": {
    "FileWrite": [
      {
        "matcher": "*.{js,ts,tsx}",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $CLAUDE_FILE"
          },
          {
            "type": "command",
            "command": "eslint --fix $CLAUDE_FILE"
          }
        ]
      }
    ],
    "FileEdit": [
      {
        "matcher": "*.test.{js,ts}",
        "hooks": [
          {
            "type": "command",
            "command": "jest $CLAUDE_FILE --coverage"
          }
        ]
      }
    ]
  }
}
效果：每次修改 JS/TS 文件，自动格式化；修改测试文件，自动运行测试。

🐛 案例 4：Bug 修复

调试类

根据错误信息定位和修复 Bug

你：

我运行 npm test 时报错，帮我看看：
Error: Cannot find module '../utils/helpers'

Claude：
Bash 运行 npm test 查看完整错误堆栈
Glob 查找 helpers 文件位置
Read 检查导入路径

问题定位：文件在 src/utils/helper.ts（单数），但导入写的是 ../utils/helpers（复数）。

Edit 修复导入路径
